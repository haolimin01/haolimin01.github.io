本博客搭建方法

2024-11-30

欢迎来到我的博客世界:)
本文章记录网站搭建的步骤，给想要拥有个人博客的同学一个参考，同时也给自己备忘。
本文章适合有一定编程基础的同学阅读，可能会给完全没有编程概念的同学造成疑惑。

开发环境

  • Platform: MacBook Pro (M1 Pro)
  • OS: macOS Sequoia
  • Editor: VS Code

hexo 安装

hexo介绍
macOS系统上一般安装有brew包管理工具，下面介绍使用brew安装hexo的步骤。

  • 安装git: brew install git
  • 安装node: brew install node (version 10.9.0)
  • 安装hexo: brew install hexo-cli -g (version 4.3.0)

创建博客

开发环境都已经准备好了，下面是博客的创建过程。

  • 项目初始化: hexo init blog (blog是我的项目名，可以改成任意自己喜欢的名字)
  • 执行命令: cd blog; npm install
  • 博客启动: hexo server 运行完这条命令后，会监听本地的4000端口，在任意浏览器网址框中输入 localhost:4000 并回车，即可看到初始的页面

配置主题

hexo主题集合 有非常多漂亮精美的主题，可以选择一个自己喜欢的主题。
本人喜欢 hexo cards，下面介绍如何接入该主题。

  • 执行命令: cd blog; git clone https://github.com/ChrAlpha/hexo-theme-cards.git themes/cards
  • 修改主题配置: 打开 blog 目录下的 _config.yml 文件，将原来的 theme: landscape 更改为 theme: cards
  • 重新执行命令 hexo server 刷新浏览器就可以看到更新后的漂亮主题
  • 在 blog/themes/cards/_config.yml 中修改某些字段，可以更改 cards 主题显示的内容，参考 字段含义

配置 GitHub 仓库

  • 在自己的GitHub主页创建仓库: yourname.github.io (注意，yourname 是自己GitHub用户名，后面的 .github.io 必须有)
  • 在 blog/_config.yml 文件最后的 deploy 字段，修改或添加如下内容
    deploy:
  type: git
  repo: git@github.com:yourname/yourname.github.io.git
  branch: master
  • 执行命令: hexo clean; hexo generate; hexo deploy 即可将本地内容推送到远程GitHub仓库中

创建标签页 (仅一次)

  1. cd blog
  2. hexo new page tags
  3. 打开 blog/source/tags/index.md 文件，在date: xxx下一行添加内容 layout: tag

创建新的推文 (post)

  1. cd blog
  2. hexo new "your-post-name"
  3. 在 blog/source/_posts 目录下可以看到 your-post-name.md 文件，这是一个 Markdown 格式的文件。Markdown是目前非常流行的一种文档格式，可以轻松的表示内容之间的层级关系以及各种修饰。
  4. 编辑完成该文件后，在 blog 目录执行 hexo server 即可在本地浏览器看到修改后的内容
  5. 确认显示没有错误之后，推送到 GitHub 端 hexo clean; hexo generate; hexo deploy
  6. 打开网页 yourname.github.io 即可看到自己编辑的内容

创建新的页面 (page)

  1. cd blog
  2. hexo new page "your-page-name"
  3. 在 blog/source 目录下可以看到多了一个文件夹 your-page-name，在 your-page-name/index.md 中编辑自己想要的内容
  4. 同样执行 hexo server 在本地看运行的效果
  5. 推送到 GitHub: hexo clean; hexo generate; hexo deploy
  6. 打开网页 yourname.github.io 即可看到自己编辑的内容
