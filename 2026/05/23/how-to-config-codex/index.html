国内环境配置和使用 Codex

2026-05-23

欢迎来到我的博客世界:)

在我的博士师弟强烈推荐下，我也尝试了现在大火的 Codex 服务，有一种相见恨晚的感觉。
本文章记录在国内网络环境下配置和使用 OpenAI Codex 的过程，给想要尝试 AI 编程助手的同学一个参考，同时也给自己备忘。

准备工作

Codex 可以理解为一个更懂工程项目的 AI 编程助手。它不只是回答几段代码，而是可以读项目、改文件、跑命令、看报错、补测试，甚至帮你做 code review。

在正式开始之前，建议先准备下面几样东西：

  • 一个稳定的网络环境
  • 一个可以接收验证码的手机号
  • 一个 OpenAI 账号
  • 一个可以完成海外线上支付的 Visa 或 Mastercard
  • 本地开发环境，例如 Git、Node.js、Python、VS Code 等

配置网络环境

国内使用 OpenAI 服务，第一关通常不是模型，而是网络。

常见问题包括：ChatGPT 打不开、OpenAI Platform 登录失败、验证码收不到、支付页面加载不出来、终端里的 Codex CLI 连不上服务等。

所以第一步先把网络环境配置好，这里需要的是可以访问全球互联网的网络，注意不是局域网环境。我就不推荐具体软件了，反正我走的是正规渠道:)

配置完成后，可以用浏览器打开下面几个网站测试。

  • OpenAI
  • ChatGPT
  • OpenAI Platform

如果这些网站都能稳定打开，再继续注册和登录账号。

如果你有 Google 账号，也可以直接用 Google 账号登录，会省掉一部分注册步骤。

手机号验证

最近注册或登录 OpenAI 账号时，经常会遇到手机号验证。由于你懂的原因，国内 +86 手机号通常无法正常接收验证码。

我自己尝试过一些海外短信服务。比如在 Hero SMS 上试过越南、巴拿马、加拿大等地区，结果都失败了，最后使用泰国手机号才成功收到验证码。

添加 Codex 数字验证码

账号能登录之后，建议马上开启双重验证。这里说的数字验证码，就是认证器 App 里每隔几十秒刷新一次的 6 位验证码，也叫 TOTP。

大致流程如下。

  1. 登录 ChatGPT 或 OpenAI Platform
  2. 进入账号安全设置
  3. 找到双重验证或 Multi-factor authentication
  4. 使用认证器 App 扫描二维码
  5. 输入 App 中显示的 6 位数字验证码
  6. 保存备用恢复码

认证器 App 可以使用 Google Authenticator、Microsoft Authenticator、1Password、Bitwarden 等。我自己用的是 Microsoft Authenticator。配置完成之后，后续登录可以直接用 6 位验证码验证，不用每次都依赖手机号。

恢复码一定要保存好，最好放在密码管理器里。换手机之前也要确认认证器已经迁移，不然很可能把自己锁在门外。

配置支付方式

如果只是体验免费功能，可以先跳过这一步。如果要使用付费版 ChatGPT 或者更完整地使用 Codex，就需要绑定支付方式。

比较常见的是 Visa 或 Mastercard 信用卡。建议优先使用自己真实持有、可以正常海外线上支付的卡片。账单国家、姓名、邮箱这些信息尽量保持一致，不然容易出现支付失败、订阅开通失败、反复验证等问题。

如果普通信用卡支付失败，可以检查下面几项。

  • 卡片是否支持海外线上支付
  • 银行是否拦截了外币交易
  • 账单地址是否填写正确
  • 当前网络环境是否稳定
  • 是否短时间内重复提交了太多次

国内可以在 某橙色软件 上购买 Visa 信用卡服务。我购买的是 Plus 会员，一个月 20 美元。

Codex 的使用方式

Codex 最适合处理真实工程里的问题。它不是只会回答“这段代码是什么意思”，而是更像一个可以一起干活的开发同伴。

比较适合交给它的任务包括：

  • 阅读项目结构，解释某个模块的作用
  • 根据报错定位问题
  • 修改一个小功能
  • 补充测试用例
  • 做 code review
  • 重构重复逻辑
  • 写脚本处理工程杂活
  • 帮忙生成 README 或使用文档

Codex 带来的思考

第一次使用 Codex，你就能明显感受到它和 豆包、通义千问、腾讯混元 这类普通聊天助手的不同。

你可以直接说想做什么、想达到什么效果，不需要一开始就把实现方式想得非常清楚。Codex 会自己读项目、拆任务、修改代码、运行命令，然后把结果反馈给你。

以前完成一个项目，首先要想需求，然后到处搜索教程，搭建编程环境，再开始没日没夜地实现。在这个过程中，你可能要会前端、后端、算法、美工、测试、部署等一堆东西。

一个人要干大量繁琐的工作，而需求往往变化又很快。这也就是产品经理和程序员经常吵架的原因:)

有了 Codex 之后，工作方式会发生变化。你不再只是盯着每一行代码怎么写，而是更多地思考目标是什么、用户是谁、怎么推广、怎么盈利、怎么验证结果。

这其实是一个很大的转变：从打工者思维，慢慢切换到老板思维。

购买 Codex 服务后，我花一天时间完成了 CNN 网络搭建、训练数据准备、训练过程 loss 分析、根据训练结果优化模型等工作。以前这些事情也能做，但迭代速度完全不一样。

正如你所见，本篇博客的大部分内容也是使用 Codex 完成的。

总结

国内环境使用 Codex，真正麻烦的地方往往不是 Codex 本身，而是账号、网络、手机号、支付这些前置条件。只要前面配置稳定下来，后面使用体验就会非常顺滑。

配置好之后，Codex 会明显改变写代码的节奏。以前需要自己慢慢翻项目、查报错、补测试，现在很多重复性的工程劳动都可以交给它。开发者要做的事情，也从“逐行搬砖”更多地变成了“定义问题、审查结果、控制质量”。

这大概就是我喜欢 Codex 的地方：它不是替我写完所有代码，而是让我更快进入真正需要思考的部分。

努力做第一个吃螃蟹的人吧！
